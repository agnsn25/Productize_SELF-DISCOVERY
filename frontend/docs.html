Self-Discover API

A reasoning framework that discovers problem-specific cognitive strategies before solving. Based on Google DeepMind's SELF-DISCOVER paper.

Core idea: Instead of using one-size-fits-all prompting, the model first analyzes your problem type and builds a custom reasoning structure. That structure is then reused to solve any problem of the same type — faster and more accurately.

Auditable Reasoning

Every answer comes with a full reasoning trace — you can see exactly how the model thought through the problem.

Compute Efficient

Discovery runs once per problem type. After that, inference is a single pass using the saved structure — up to 10-40x cheaper than ensemble methods.

Better Accuracy

Problem-specific reasoning outperforms generic Chain-of-Thought on complex tasks — up to 32% improvement on benchmarks.

Quick Start

Two steps to go from problem to answer.

1

Discover a reasoning structure

Describe a type of problem (not a specific instance). The API analyzes it and builds a reusable reasoning structure.

bash 
curl -X POST http://localhost:8000/api/discover \
  -H "Content-Type: application/json" \
  -d '{
    "task_description": "Solve multi-step math word problems involving rates, percentages, and unit conversions"
  }'

Save the id from the response — that's your structure ID.

2

Solve problems using that structure

Pass the structure ID and a specific problem. The model follows the discovered reasoning steps to solve it.

bash 
curl -X POST http://localhost:8000/api/infer \
  -H "Content-Type: application/json" \
  -d '{
    "structure_id": "YOUR_STRUCTURE_ID",
    "problem": "A store offers a 20% discount on a $150 jacket, then an additional 15% member discount. Sales tax is 8.5%. What is the final price?"
  }'

You can reuse the same structure ID for any number of problems of this type.

How It Works

SELF-DISCOVER has two stages, each with distinct roles.

Stage 1 — Discovery

This runs once per problem type. Three sequential LLM calls build a reasoning structure:

SELECT From 39 reasoning modules, pick the ones most relevant to this problem type
ADAPT Rephrase each selected module to be specific and actionable for this problem type
IMPLEMENT Compose the adapted modules into a structured, step-by-step JSON reasoning plan
Structure saved with a unique ID
Stage 2 — Inference

This runs per problem. One LLM call fills in the reasoning structure with the specific problem's details.

SOLVE Follow the saved structure step-by-step, applying each reasoning instruction to the new problem

Reasoning Modules

The framework draws from a library of 39 cognitive reasoning modules — derived from the original SELF-DISCOVER paper. During the SELECT step, the model picks the most relevant ones for your problem type.

These modules fall into a few broad categories:

Problem Decomposition

  • How can I break down this problem into smaller, more manageable parts?
  • How can I simplify the problem so that it is easier to solve?
  • What is the core issue or problem that needs to be addressed?
  • Let's make a step by step plan and implement it with good notation and target.

Thinking Styles

  • Critical Thinking: Analyze from different perspectives, question assumptions, evaluate evidence.
  • Try creative thinking, generate innovative and out-of-the-box ideas.
  • Use systems thinking: Consider the problem as part of a larger system.
  • Use Reflective Thinking: Step back, examine biases and mental models.

Problem Classification

  • Does the problem involve making decisions or choices?
  • Does the problem involve a sequence of steps or procedures?
  • Does the problem involve quantitative reasoning or data analysis?
  • Does the problem involve abstract thinking or conceptual understanding?

Analysis & Evaluation

  • What are the key assumptions underlying this problem?
  • What are the potential risks and drawbacks of each solution?
  • Use Risk Analysis: Evaluate potential risks, uncertainties, and tradeoffs.
  • How could I measure progress on this problem?
The full list contains 39 modules. You don't pick them — the model does during the SELECT step. But knowing they exist helps you write better task descriptions.

Discovery Pipeline

Here's what happens under the hood when you call /api/discover:

1

SELECT

The LLM receives your task description and the full list of 39 reasoning modules. It picks the subset that's genuinely useful — typically 4-8 modules. The prompt explicitly says "do not select all of them" to avoid noise.

2

ADAPT

Each selected module gets rephrased to be specific to your task. For example, "Break the problem into smaller parts" might become "Break the word problem into individual calculation steps, isolating each rate or percentage operation."

3

IMPLEMENT

The adapted modules are composed into a structured JSON plan with ordered steps. Each step has an instruction and the reasoning_module it's based on. The final step always synthesizes into a conclusion.

All three steps use Gemini 2.5 Pro with extended thinking enabled. The thinking traces from each step are saved alongside the structure for full auditability.

Compare Mode

The /api/infer/compare endpoint runs two approaches in parallel on the same problem:

Naive

A straightforward "solve this step by step" prompt. No structure, no module selection. This is how most LLM apps work today.

vs
SELF-DISCOVER

Uses the pre-built reasoning structure to guide the model through problem-specific steps. The model fills in each step with its reasoning.

Both run concurrently via asyncio.gather, so you're not waiting twice. Compare mode is useful for:

  • Validating that a discovered structure actually improves answers for your problem type
  • Demos and presentations showing the value of structured reasoning
  • Iterating on task descriptions — if SELF-DISCOVER isn't beating naive, your task description might be too vague

Built on SELF-DISCOVER: Large Language Models Self-Compose Reasoning Structures (Zhou et al., 2024)
